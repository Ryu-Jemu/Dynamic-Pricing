# Dynamic Pricing Optimization for 5G Network Slicing via Three-Part Tariff

## 모델 개요

이 모델은 **5G 네트워크 슬라이싱** 환경에서 **강화학습(RL) 에이전트**가 **Three-Part Tariff(3PT)** 가격을 동적으로 조정하여 서비스 제공자의 수익을 극대화하는 시스템이다.

```
┌─────────────────────────────────────────────────────────────────┐
│                        핵심 아이디어                              │
│                                                                 │
│  RL 에이전트가 매 시간(timestep)마다 URLLC와 eMBB 슬라이스의       │
│  고정요금(F)과 초과요금(p)을 결정 → 가입자 이탈/유입, 사용량,      │
│  QoS에 따른 수익과 패널티를 관찰 → 장기 누적 수익을 최대화하는     │
│  가격 정책(π)을 학습한다.                                        │
└─────────────────────────────────────────────────────────────────┘
```

## 2. Three-Part Tariff (3PT) 요금 구조

사용자 i의 슬라이스 s에서 시간 t의 청구액:

```
B_{i,s,t} = F_{s,t} + max(0, q_{i,s,t} - Q̄_s) × p_{s,t}
             ───────   ──────────────────────────────────
             고정요금          초과사용분 × 초과요금
```

### 세 가지 구성요소

```
요금 = ① 고정요금(F) + ② 허용량(Q̄) 내 무료 + ③ 초과분 × 초과요금(p)

예시 (eMBB):
  F = $30, Q̄ = 30 GB, p = $5/GB, 실제 사용량 q = 42 GB

  청구액 = $30 + max(0, 42 - 30) × $5
         = $30 + 12 × $5
         = $30 + $60 = $90

  → 허용량 이내 사용자: $30만 지불 (고정)
  → 허용량 초과 사용자: $30 + 초과분 × $5
```

### RL 에이전트가 학습하는 것

- **F_{s,t}** (고정요금): 슬라이스별 매 시간 결정 ← 학습 대상
- **p_{s,t}** (초과요금): 슬라이스별 매 시간 결정 ← 학습 대상
- **Q̄_s** (허용량): 슬라이스별 고정 상수 (학습하지 않음)

→ 에이전트의 **액션**: `a_t = (F_U, p_U, F_E, p_E)` — 4차원 연속값

## 3. 데이터 사용량 모델

각 사용자의 시간당 데이터 사용량은 **로그정규분포**를 따른다:

```
q_{i,s,t} ~ LogNormal(μ_s, σ²_s)

이유: 18년간의 인터넷 트래픽 실증 연구에서 로그정규분포가
      가장 정확한 모델임이 확인됨 (Alasmar et al. 2021)
```

### 직관적 이해

```
로그정규분포의 특성:
  - 대부분의 사용자는 평균 이하의 데이터를 사용
  - 소수의 "헤비 유저"가 매우 많은 데이터를 사용 (긴 꼬리)

URLLC:   ████████████▓▓▓░░░         평균 3.49 GB/hr
         대부분 적은 사용량 ──→ 일부 큰 사용량

eMBB:    ████████████████████▓▓▓▓▓░░░░░░   평균 29.96 GB/hr
         넓게 퍼진 분포 ──→ 더 긴 꼬리
```

### 허용량 초과 확률

| 슬라이스 | Q̄ | P(q > Q̄) | 의미 |
|---------|------|-----------|------|
| URLLC | 5 GB/hr | 19.44% | 약 5명 중 1명이 초과 |
| eMBB | 30 GB/hr | 32.69% | 약 3명 중 1명이 초과 |

## 4. 타임스텝 동역학 (매 시간 반복)

하나의 타임스텝은 4개 Phase로 구성된다:

```
┌─────────────────────────────────────────────────────────────────────┐
│                     하나의 타임스텝 (1시간)                           │
│                                                                     │
│  Phase A          Phase B              Phase C           Phase D    │
│  에이전트 결정     가입자 동역학          사용량 & QoS      학습 신호  │
│                                                                     │
│  ┌──────────┐    ┌──────────────┐    ┌────────────┐   ┌──────────┐ │
│  │ 1. 상태  │    │ 3. 이탈      │    │ 5. 사용량  │   │ 7. 보상  │ │
│  │    관찰  │───→│    (기존고객) │───→│    생성 +  │──→│    계산  │ │
│  │          │    │              │    │    청구    │   │          │ │
│  │ 2. 가격  │    │ 4. 유입      │    │            │   │ 8. 상태  │ │
│  │    설정  │    │    (신규고객) │    │ 6. QoS    │   │    전이  │ │
│  └──────────┘    └──────────────┘    │    실현    │   └──────────┘ │
│                                      └────────────┘                 │
│                                                                     │
│  RL Agent          Environment          Environment     Environment │
└─────────────────────────────────────────────────────────────────────┘
```

### Phase A — 에이전트 결정

**Step 1. 상태 관찰**
```
S_t = (N_{U,t}, N_{E,t}, η_{U,t-1}, η_{E,t-1})
       ─────── ─────── ──────────  ──────────
       URLLC   eMBB    이전시간     이전시간
       가입자수 가입자수 URLLC QoS  eMBB QoS
```

**Step 2. 가격 설정**
```
a_t = (F_{U,t}, p_{U,t}, F_{E,t}, p_{E,t})
       ─────── ─────── ─────── ───────
       URLLC   URLLC   eMBB    eMBB
       고정요금 초과요금 고정요금 초과요금
```

### Phase B — 가입자 동역학

**Step 3. 이탈 (기존 고객의 해지 결정)**

각 기존 가입자가 독립적으로 이탈할 확률:

```
P^dep = σ(γ₀ + γ_F × F̃ + γ_p × p̃ - γ_η × η_{t-1})
        ──   ─────────   ─────────   ──────────────
        기본  가격이 높으면  초과요금이   QoS가 좋으면
        유지  이탈 증가↑    높으면↑      이탈 감소↓

σ(x) = 1/(1+e^{-x})  ← 시그모이드 함수 (출력: 0~1 확률)

핵심 포인트:
  - 가격(F, p)이 높을수록 → 이탈 확률 증가
  - 이전 시간 QoS(η_{t-1})가 좋을수록 → 이탈 확률 감소
  - URLLC 사용자는 QoS에 6배 더 민감 (γ_{η,U}=3.0 vs γ_{η,E}=0.5)
```

**Step 4. 유입 (신규 고객의 가입 결정)**

```
P^arr = σ(β₀ - β_F × F̃ - β_p × p̃)
                ────────   ────────
                가격 높으면  초과요금 높으면
                유입 감소↓   유입 감소↓

신규 가입자 수 ~ Poisson(λ_max × P^arr)

핵심 포인트:
  - 신규 고객은 가격만 보고 결정 (QoS 경험 없음)
  - Poisson 분포: 시간당 평균 유입률에 무작위성 추가
```

**Step 5. 활성 사용자 확정**
```
N^active = N^surv (잔존자) + N^new (신규 유입)

이 사용자들이 이번 시간의 서비스를 이용한다.
```

### Phase C — 사용량과 QoS

**Step 5. 사용량 생성 + 청구**
```
각 활성 사용자마다:
  1) q ~ LogNormal(μ, σ²)  ← 사용량 생성
  2) B = F + max(0, q - Q̄) × p  ← 청구액 계산
```

**Step 6. QoS 실현**
```
η_{s,t} ~ Uniform(η^low, η^high)  ← 네트워크 상태에 따른 무작위 QoS

QoS는 외생적 (에이전트가 직접 제어 불가)
→ 에이전트는 QoS 변동에 "반응적으로" 가격을 조정
   (QoS 나빠지면 → 이탈 방지를 위해 가격 인하 등)
```

### Phase D — 학습 신호

**Step 7. 보상 계산**
```
수익:   R_t = Σ (모든 슬라이스, 모든 활성 사용자의 청구액 합계)

패널티: Pen_t = Σ w_s × N^active × max(0, η^tgt - η_{s,t})
                ───   ────────   ─────────────────────────
                가중치  사용자수   QoS 목표 미달분 (0이면 패널티 없음)

보상:   r_t = R_t - Pen_t
              ───   ──────
              수익   QoS 위반 벌금

→ 에이전트는 "수익 극대화"와 "QoS 유지" 사이의 균형을 학습
```

**Step 8. 상태 전이**
```
N_{s,t+1} = N^active_{s,t}       ← 활성 사용자가 다음 시간의 가입자
η_{s,(t+1)-1} = η_{s,t}          ← 이번 QoS가 다음 시간의 이탈 결정에 사용

S_{t+1} = (N_{U,t+1}, N_{E,t+1}, η_{U,t}, η_{E,t})
           ──────────────────── ────────────────
           업데이트된 가입자 수      이월된 QoS
```

## 5. MDP (Markov Decision Process) 정의

강화학습의 수학적 프레임워크:

```
┌──────────┬──────────────────────────────────────────────────────┐
│ 구성요소  │ 정의                                                │
├──────────┼──────────────────────────────────────────────────────┤
│ 상태(S)  │ (N_U/N_U₀, N_E/N_E₀, η_{U,prev}, η_{E,prev}) — 4차원│
│          │ ※ 가입자수는 초기값으로 정규화 (스케일 균형)           │
│ 액션(A)  │ (F_U, p_U, F_E, p_E) — 4차원 연속                  │
│ 보상(R)  │ r_t = (수익 - QoS 패널티) × 10⁻⁵  (학습 안정성)     │
│ 전이(P)  │ Phase B→C→D의 확률적 결과                            │
│ 초기상태  │ (N_U=1000, N_E=5000, η_U=0.95, η_E=0.90)          │
│ 목표     │ max_π E[Σ γ^{t-1} × r_t], γ=0.99                   │
│ 시간범위  │ T=720 스텝 (1개월, 시간 단위)                        │
└──────────┴──────────────────────────────────────────────────────┘
```

### 에이전트의 딜레마

```
높은 가격 설정 시:
  ✅ 단기 수익 증가 (높은 청구액)
  ❌ 이탈 증가, 유입 감소 (장기적 가입자 감소)
  ❌ 가입자 감소 → 미래 수익 감소

낮은 가격 설정 시:
  ❌ 단기 수익 감소
  ✅ 이탈 감소, 유입 증가 (가입자 유지/성장)
  ✅ 가입자 증가 → 미래 수익 증가

QoS가 나빠졌을 때:
  → URLLC 사용자의 이탈 급증 위험 (QoS 민감도 6배)
  → 에이전트: 가격을 낮춰 이탈 방지? or 유지하여 수익 확보?

이 다차원적 트레이드오프를 RL이 데이터 기반으로 학습한다.
```

## 6. 강화학습 알고리즘

### SAC (Soft Actor-Critic) — 주력 알고리즘

```
특징:
  - 연속 액션 공간에 최적화 (4D 가격 연속 조정)
  - 엔트로피 정규화: 탐색(exploration)과 활용(exploitation) 자동 균형
  - Twin Q-네트워크: 과대평가 방지
  - Off-policy: 경험 재활용 → 학습 효율성

구조:
  ┌─────────────┐    ┌─────────────────┐
  │ Actor (π)    │    │ Critic (Q₁, Q₂) │
  │ 상태→가격분포│    │ (상태,가격)→가치 │
  │ [256-256-256]│    │ [256-256-256]    │
  └─────────────┘    └─────────────────┘

학습 목표:
  max  E[Q(s,a)] + α × H(π)
       ─────────   ──────────
       기대 수익    엔트로피 보너스
       극대화      (다양한 가격 시도 장려)
```

### PPO (Proximal Policy Optimization) — 비교 알고리즘

```
특징:
  - On-policy: 안정적이지만 데이터 효율 낮음
  - Clipped surrogate: 정책 급변 방지
  - SAC 대비 수렴은 안정적이나 느림
```

## 7. 전체 학습 과정

```
┌────────────────────────────────────────────────────────────────┐
│                    전체 학습 파이프라인                           │
│                                                                │
│  Step 1: 환경 초기화                                            │
│    N_U=1000, N_E=5000, η_U=0.95, η_E=0.90                    │
│                                                                │
│  Step 2: 에피소드 반복 (2000 에피소드)                           │
│    ┌─────────────────────────────────────────────┐             │
│    │  각 에피소드 (720 타임스텝 = 1개월):          │             │
│    │                                             │             │
│    │  for t = 1 to 720:                          │             │
│    │    1) 상태 S_t 관찰                          │             │
│    │    2) 정책 π로 가격 a_t 결정                  │             │
│    │    3) 환경에 a_t 적용 → 이탈/유입/사용/QoS    │             │
│    │    4) 보상 r_t = 수익 - 패널티 수신           │             │
│    │    5) (S_t, a_t, r_t, S_{t+1}) 저장         │             │
│    │    6) 정책 업데이트 (SAC/PPO)                │             │
│    │                                             │             │
│    │  에피소드 총 보상 기록                        │             │
│    └─────────────────────────────────────────────┘             │
│                                                                │
│  Step 3: 평가                                                  │
│    - 학습된 정책 고정                                           │
│    - 20 에피소드 평가, 5 시드 반복                               │
│    - 평균 ± 95% CI 보고                                        │
│                                                                │
│  Step 4: 베이스라인 비교                                        │
│    - Static-Oracle, Static-Heuristic                           │
│    - 2PT-SAC, Flat-SAC, Myopic-SAC, Random                    │
└────────────────────────────────────────────────────────────────┘
```

## 8. 검증된 핵심 수치

6중 Audit (수학+문헌+파라미터+수치시뮬+경제합리성+수식유도)를 통해 검증 완료.

### 기대 청구액

```
URLLC 사용자 1인의 시간당 기대 청구액:
  E[B_U] = $50 + E[max(0, q-5)] × $10
         = $50 + 0.5586 × $10
         = $55.59

eMBB 사용자 1인의 시간당 기대 청구액:
  E[B_E] = $30 + E[max(0, q-30)] × $5
         = $30 + 10.3342 × $5
         = $81.67
```

### 정상상태 분석

```
기준 가격에서의 시스템 균형:

  이탈확률: P^dep = σ(-10.77) = 2.10 × 10⁻⁵/hr → 월 1.5% 이탈
  
  URLLC 유입: 0.05 × 0.6457 = 0.0323/hr (월 ~23명)
  eMBB 유입:  0.15 × 0.7503 = 0.1125/hr (월 ~81명)
  
  정상상태 가입자:
    N*_U ≈ 1,536명 (초기 1,000명에서 점진적 성장)
    N*_E ≈ 5,354명 (초기 5,000명에서 점진적 성장)
```

### 보상 구성

```
시간당 기대 보상 (기준가격, 초기 가입자 기준):

  수익:  R = 1000 × $55.59 + 5000 × $81.67 = $463,940
  패널티: Pen = 500 × 1000 × 0.04999 + 50 × 5000 × 0.025 = $31,245
  보상:  r = $463,940 - $31,245 = $432,695

  패널티/수익 비율 = 6.7% → RL에게 유의미한 학습 신호
```
